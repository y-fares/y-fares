## How I approach projects

I usually start from a simple question:

> What painful manual workflow can be turned into a useful AI/data product?

Then I build around four principles:

1. **Business value first** ã no technology for decoration.
2. **Usable interface** ã a tool must be understandable by real users.
3. **Traceable outputs** ã especially for AI/RAG systems, sources and structure matter.
4. **Deployable architecture** ã clean configuration, APIs, tests and documentation when relevant.
