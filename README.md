## What I do

I come from a strong **Data / BI / Big Data / Product** background and now focus on building useful AI applications with clear business value.

My work is not about making toy demos. I care about tools that help users:

* understand complex documents faster;
* prepare better decisions;
* extract actions from messy information;
* connect AI capabilities with real product workflows.

## How I approach projects

I usually start from a simple question:

> What painful manual workflow can be turned into a useful AI/data product?

Then I build around four principles:

1. **Business value first** — no technology for decoration.
2. **Usable interface** — a tool must be understandable by real users.
3. **Traceable outputs** — especially for AI/RAG systems, sources and structure matter.
4. **Deployable architecture** — clean configuration, APIs, tests and documentation when relevant.
