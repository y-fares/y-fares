How I approach projects

I usually start from one question: what painful manual workflow can be turned into a useful AI/data product?

1. Business value first

No technology for decoration.

2. Usable interface

A tool must be understandable by real users.

3. Traceable outputs

Sources and structure matter, especially for AI.

4. Deployable architecture

Clean config, APIs, tests and documentation when relevant.
